# CodexNotch 功能演示视频指南

这份文档用于以后录制 60–90 秒产品演示。目标不是逐项念功能，而是讲清楚一个场景：**用户离开 Codex 窗口后，仍能掌握任务进度并快速回来。**

## 1. 推荐成片规格

- 主版本：16:9，1920×1080，60fps，约 75 秒。
- 社交媒体裁切：9:16，1080×1920，约 30 秒。
- 录制范围：MacBook 整个屏幕，确保实体刘海和插件相对位置可见。
- 鼠标：开启点击提示，但不要使用过大的彩色光圈。
- 音频：轻量环境电子乐；点击、展开和完成可加入很轻的触感音效。

## 2. 录制前准备

- 隐藏通知、邮件、私人项目名称和菜单栏敏感信息。
- 准备 3 个没有隐私内容的 Codex 示例任务。
- 将剩余用量保持在容易辨认的数值，例如 73%。
- 清除旧的待查看任务，确保录制从空闲状态开始。
- 将桌面背景换成低对比度纯色或渐变。
- 关闭其他刘海工具，除非专门演示共存功能。

## 3. 示例任务

1. `分析这个 README，并给出三个最重要的改进建议。`
2. `检查这个 Swift 文件是否存在状态同步问题。`
3. 拖入一个无隐私的图片，要求 Codex 描述界面层级。

## 4. 75 秒分镜

| 时间 | 画面 | 操作 | 字幕/旁白 |
| --- | --- | --- | --- |
| 0–5s | 干净桌面与实体刘海，插件处于空闲 | 不操作 | “Codex 在工作时，你不必一直留在它的窗口。” |
| 5–13s | Codex 中启动一个任务，立刻切到浏览器 | 启动任务后切换应用 | “CodexNotch 把任务状态留在视线顶部。” |
| 13–22s | 刘海显示任务名、分析状态、耗时与模型 | 鼠标短暂停留在任务上 | “状态、时间和模型，一眼就够。” |
| 22–34s | 再启动两个任务，顶部显示 `+N` | 点击任务展开，再次点击收回 | “多任务默认保持安静，需要时再展开。” |
| 34–45s | 将示例文件拖到刘海 | 松开并点击“在 Codex 新建对话” | “文件、网址和文字，可以直接拖进去开始分析。” |
| 45–55s | 空闲悬停后的用量页面 | 展示今日 Token、余量、重置和版本 | “用量与重置时间，也不用再到处找。” |
| 55–66s | 任务完成，底部提示自然出现 | 点击绿色确认按钮 | “完成后，点一下就回到对应对话。” |
| 66–75s | 回到干净桌面，显示图标和名称 | 无操作 | “CodexNotch。Codex 状态，恰好在顶部。” |

## 5. 30 秒短版

- 0–4s：从 Codex 切到其他应用，顶部任务仍在运行。
- 4–10s：展示任务耗时和模型。
- 10–16s：点击 `+N` 展开三个任务。
- 16–22s：拖入文件并新建 Codex 对话。
- 22–27s：完成提示出现，点击返回对话。
- 27–30s：图标、产品名和一句定位。

## 6. 镜头原则

- 每个状态至少停留 1.5 秒，避免观众看不清文字。
- 展开和收起必须完整录到，不要在动画中间剪切。
- 鼠标移入前预留半秒，让默认状态先被看见。
- 避免快速来回悬停，这会让产品显得不稳定。
- 任务完成提示出现时，不要立刻点击，至少展示 2 秒。
- 如果录到顶部裂隙、错位、遮挡或动画掉帧，应重新录制该镜头。

## 7. 建议字幕

- “离开 Codex，进度仍然可见。”
- “多任务，默认只占一行。”
- “拖入内容，直接开始新对话。”
- “完成提醒，点击返回对应任务。”
- “本机读取，不上传会话内容。”

## 8. 录完后的验收清单

- [ ] 没有出现真实姓名、Token、私人文件路径或未公开项目。
- [ ] 实体刘海与插件边缘对齐清楚。
- [ ] 至少展示一次默认、运行、多任务、拖入和完成状态。
- [ ] 所有展开与收起动画完整且统一。
- [ ] 字幕没有挡住刘海和任务内容。
- [ ] 成片开头 5 秒内解释产品价值。
- [ ] 结尾包含产品名 `CodexNotch` 和仓库/主页地址。
