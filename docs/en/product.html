CodexNotch

I'm Henry恒宇. I made CodexNotch because after handing work to Codex, I would move on to something else—then keep switching back just to check its progress.

CodexNotch keeps the few states that matter next to the MacBook notch: still working, waiting for you, or ready to review. It is not another workspace to manage, just a quiet way back to the right Codex conversation.

How it fits into your day

  • Stay in your browser, document, or design tool while Codex works.
  • See task title, elapsed time, model, reasoning level, and tokens at a glance.
  • Return to the matching conversation when Codex needs input or finishes.
  • Hover while idle to see today's usage, remaining capacity, reset time, and version.
  • Drop a web page, GitHub Skill, image, or file, choose the next action, then start the matching Codex conversation.
  • Search recent local tasks or review a scheduled Token report with a 14-day usage heatmap.

Install

Download the Apple silicon DMG and move the app to Applications. CodexNotch requires macOS 14+ and Codex installed locally.

This is still a preview and is not Apple-notarized. Right-click the app and choose Open on first launch.

Local by design

CodexNotch reads local Codex sessions and desktop logs to understand status. It does not upload conversations, task details, search indexes, or usage data. Dropped files are passed as local paths and analysis prompts, not uploaded by CodexNotch.

Current limits

Status detection depends on Codex's current local log format and may need updates when Codex changes. The preview is Apple silicon only, and task navigation or local file access can still depend on Codex permissions.
