# 发布新版本

## 日常更新

先正常修改、验证并提交代码：

```sh
git add <本次修改的文件>
git commit -m "说明本次更新"
git push origin main
```

## 创建可下载安装的新版本

1. 在仓库根目录新建版本说明，例如 `release-notes.md`。
2. 确保 `main` 分支没有尚未提交的修改。
3. 执行：

```sh
sh release.sh 0.1.3 release-notes.md
```

脚本会自动：

- 将应用版本改为 `0.1.3`，并增加构建号。
- 更新宣传页中的 DMG 文件名。
- 构建并签名 `CodexNotch.app`。
- 生成 `dist/CodexNotch-v0.1.3-arm64.dmg`。
- 提交并推送版本号。
- 创建 GitHub Release 并上传 DMG。

发布完成后，旧版 CodexNotch 会在下一次检查时显示“可更新 0.1.3”。应用启动后会检查一次，之后每 24 小时最多自动检查一次；也可以在设置中手动检查。

## 版本号怎么定

- `0.1.2 → 0.1.3`：修复问题、小功能。
- `0.1.3 → 0.2.0`：加入一组明显的新能力。
- `0.x → 1.0.0`：完成签名、公证并准备作为稳定版发布。

发布脚本不会自动替你编写更新说明，也不会在工作区有未提交内容时继续执行。
