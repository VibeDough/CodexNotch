# Changelog

## v0.1.3 — 2026-07-23

### 新增

- 增加本地最近任务搜索，可按任务名称、项目与摘要筛选并返回对应 Codex 对话。
- 增加轻量自动更新检查，每 24 小时最多请求一次 GitHub Release。
- 设置页增加手动检查和新版本下载入口。
- 设置改为“通用 / 功能 / 关于”横向分类，并增加官方网站与 GitHub 入口。
- 增加一键发版脚本及维护者发布说明。
- 增加本地 Token 日报、14 天圆环热力、近 7 日个人基线评价与能量轨迹图。
- 日报增加连续使用天数、14 天活跃进度，以及高强度日期的一次性轻量庆祝动画。
- 每天 18:30 后在首次空闲时提供 8 秒轻提醒，并支持从用量面板随时查看。
- 拖入网页、GitHub Skill、图片或普通文件后，根据内容提供不同的下一步操作。
- 额度首次低于 50% 时增加一次性橙色提醒，随后只保留颜色状态。

### 修复与优化

- 重启后优先恢复最近一次有效额度，避免等待新 Token 事件期间显示为 0。
- 所有新增面板继续使用固定顶部画布和统一过渡，避免设置、日报与拖入状态切换时出现裂隙。
- 移除日报提醒的内部预览启动入口，避免正式构建残留调试状态。

## v0.1.2 — 2026-07-21

### 新增

- 增加计划模式“需要用户输入”状态，使用橙色提醒并支持点击返回对应 Codex 对话。
- 根据 `request_user_input` 的 `call_id` 跟踪多轮提问，全部回答后自动恢复分析状态。
- 增加可记忆的任务整体收起按钮，收起后继续显示余量、任务阶段和执行数量。

### 修复与优化

- 区分“需要用户输入”和权限确认，避免误显示绿色确认操作。
- 收起与展开继续复用统一尺寸及边框动画管线。
- 根据当前 MacBook 的真实刘海安全区域调整收起态，中间保留 220pt，避免运行状态被摄像头遮挡。
- 保留旧版顶层输入事件兼容，适应不同 Codex 日志结构。

## v0.1.1 — 2026-07-21

### 新增

- 增加中文、英文和跟随系统语言切换。
- 增加 BoringNotch、NotchNook 冲突检测与隐藏确认。
- 增加绘图任务状态监测，避免大型图片日志导致任务状态丢失。
- 增加任务健康提示：长时间没有新事件时显示橙色状态。
- 增加今日 Token 的主要模型用量拆分。
- 空闲悬停区域支持点击空白位置快速打开 Codex。

### 修复与优化

- 修复设置展开引起的顶部裂隙和外层面板重排。
- 修复额度重置后剩余百分比更新延迟。
- 修复重置倒计时被版本信息挤压的问题。
- 修复模型上下文未被正确识别的问题。
- 减少会话文件的重复枚举，降低持续监测开销。
